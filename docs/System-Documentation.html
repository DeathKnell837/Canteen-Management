1. System Overview

The Canteen Management System is a full-stack web application designed to digitize and streamline canteen operations. It allows customers to browse the menu, place orders, and pay via a digital wallet, while administrators and staff can manage the menu, process orders through a status workflow, and track inventory.

4. Order Status Lifecycle

Orders pass through the following statuses. The database enum stores the raw value; the UI displays the friendly label shown below.

DB ValueUI LabelTriggered ByDescription
PENDINGPendingSystem (on order create)Order created, awaiting payment
CONFIRMEDPaidSystem (on payment success)Payment received, waiting for staff to start
PREPARINGPreparingAdmin/Staff → "Start Preparing"Kitchen is preparing the order
READYReady for PickupAdmin/Staff → "Mark Ready"Order is ready at the counter
PICKED_UPCompletedAdmin/Staff → "Mark Completed"Customer has received the order
CANCELLEDCancelledCustomer (Pending/Paid only) or AdminOrder was cancelled; inventory restored

5. Database Schema Summary

The system uses PostgreSQL 16 with 9 tables. Relationships are enforced via foreign keys with cascading rules. 

users (D1)
 ├── user_id         SERIAL PRIMARY KEY
 ├── email           VARCHAR(255) UNIQUE NOT NULL
 ├── phone           VARCHAR(20)
 ├── password_hash   VARCHAR(255) NOT NULL
 ├── full_name       VARCHAR(255) NOT NULL
 ├── role            ENUM(ADMIN, STAFF, CUSTOMER, FINANCE)
 ├── status          ENUM(ACTIVE, INACTIVE, SUSPENDED)
 ├── wallet_balance  DECIMAL(10,2) DEFAULT 0.00
 └── created_at      TIMESTAMP

menu_categories (D2)
 ├── category_id     SERIAL PRIMARY KEY
 └── name, description, display_order, is_active

menu_items (D2)
 ├── item_id         SERIAL PRIMARY KEY
 ├── category_id     FK → menu_categories
 ├── name, description, base_price
 ├── is_vegetarian, prep_time_minutes, is_active
 └── image_url       VARCHAR

orders (D3)
 ├── order_id        SERIAL PRIMARY KEY
 ├── user_id         FK → users
 ├── order_number    VARCHAR UNIQUE
 ├── status          ENUM(PENDING,CONFIRMED,PREPARING,READY,PICKED_UP,DELIVERED,CANCELLED)
 ├── delivery_type   ENUM(PICKUP, DELIVERY)
 ├── subtotal, tax_amount, discount_amount, total_amount
 └── created_at

order_items (D3)
 ├── order_item_id   SERIAL PRIMARY KEY
 ├── order_id        FK → orders
 ├── item_id         FK → menu_items
 ├── quantity, base_price, subtotal
 └── created_at

payments (D4)
 ├── payment_id      SERIAL PRIMARY KEY
 ├── order_id        FK → orders
 ├── user_id         FK → users
 ├── payment_method  ENUM(WALLET, CARD, CASH, ...)
 ├── amount, status  ENUM(PENDING,SUCCESS,FAILED,REFUNDED)
 └── transaction_id

inventory (D5)
 ├── inventory_id    SERIAL PRIMARY KEY
 ├── item_id         FK → menu_items (UNIQUE)
 ├── quantity_available, reorder_level
 └── unit_cost, last_stock_check

wallet_transactions (D6)
 ├── transaction_id  SERIAL PRIMARY KEY
 ├── user_id         FK → users
 ├── amount, transaction_type
 ├── balance_after
 └── created_at

feedback
 ├── feedback_id     SERIAL PRIMARY KEY
 ├── order_id        FK → orders
 ├── user_id         FK → users
 └── rating (1-5), comment, feedback_type, status
